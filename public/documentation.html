curl-load Documentation

🚀 Getting Started

Turn any curl command into a load test in seconds.

1. Paste your curl

curl -X GET 'https://api.example.com/products'

2. Configure the test

  • Users → number of concurrent users
  • Duration → how long the test runs
  • Ramp-up → how quickly load increases

Tip: Start small (1–5 users) for real systems.

3. Run the test

Click Run Test to see:

  • Live results
  • Latency chart
  • Success / failure counts

4. Add variables (optional)

Use variables to change request values:

${variableName}

Example:

curl -X POST 'https://api.example.com/cart' \
  -H 'Content-Type: application/json' \
  -d '{"productId": "${productId}", "qty": 1}'

Define:

  • Name: productId
  • Type: Sequential
  • Values: 101, 102, 103

Each request will use the next value.

5. Compare runs

Select multiple runs and click Compare to analyze performance differences.

Variables

Variables allow dynamic values in your requests.

Syntax

${variableName}

Built-in Variables

VariableDescription
${random}Random number (0–999999)
${user}Virtual user ID
${iteration}Request number per user
${timestamp}Current timestamp
${isoTimestamp}ISO formatted time

User Variables

Each variable has:

  • Name
  • Type
  • Values

Variable Types

Constant (same value every time)

Values: 101, 102, 103

Request 1 → 101
Request 2 → 101
Request 3 → 101

Sequential (next value per request)

Values: 101, 102, 103, 104

Request 1 → 101
Request 2 → 102
Request 3 → 103
Request 4 → 104
Request 5 → 101 (loops back)

Best for: realistic data simulation

Random (random value each request)

Values: 101, 102, 103

Request 1 → 102
Request 2 → 101
Request 3 → 103

Best for: spreading load, avoiding caching

Example

curl -X GET 'https://api.example.com/products?category=${category}'

Variable:

category
Type: Random
Values:
books
electronics
clothing

Each request hits a different category.

Local vs Remote

  • Local: variables change per request
  • Remote: variables change per user and per iteration

Key Idea

  • Constant → same value
  • Sequential → next value
  • Random → different value

Variables control how your data changes across requests.

⚠️ Safety: This tool generates load. Be careful when running against production systems.
